1 Évolutions vs v1 (DynamoDB)

3 changements majeurs : (1) Migration DynamoDB → Aurora PostgreSQL : le SQL devient naturel avec les multi-tarifs et les calculs d'agrégats. (2) La table tarif_personne remplace l'unique prix_par_personne_nuit — on peut en avoir autant que voulu. (3) Un nouveau rôle Trésorier gère les tarifs et la configuration globale du site.

3 Règles de gestion multi-tarifs

Principe général

Un séjour peut avoir N catégories de personnes, chacune avec son propre tarif/nuit. Exemples : "Membre de l'union" à 14 €, "Groupe de jeunes" à 12 €, "Extérieur" à 18 €. Le resp. location choisit les catégories et leurs effectifs prévus au moment de la création du séjour. Le gardien saisit les effectifs réels par catégorie.

Calcul du minimum et de nb_facturees :
Le minimum (ex. 40) est global au séjour (sejour.min_personnes_total). Il porte sur la somme de toutes les catégories.
→ total_reel = SUM(sejour_categorie.nb_reelles)
→ Si total_reel ≥ min : chaque catégorie est facturée à ses nb_reelles.
→ Si total_reel < min : on ajoute une ligne COMPLEMENT_MINIMUM calculée au tarif moyen pondéré, ou on distribue l'écart proportionnellement. Recommandation : ligne complément — plus lisible sur la facture.
Exemple de séjourCatégorieTarifPrésents (réel)FacturésMontant (7 nuits)
Famille Dupont
38 présents, min 40		 Membre de l'union 14,00 €/nuit 25 25 25 × 14 × 7 = 2 450 €
Groupe de jeunes 12,00 €/nuit 13 13 13 × 12 × 7 = 1 092 €
Complément minimum 13,24 € moy. — 2 (= 40 − 38) 2 × 13,24 × 7 = 185 €
Total hébergement 40 pers. facturées 3 727 €
Snapshot obligatoire : les champs nom_snapshot et prix_nuit_snapshot dans sejour_categorie copient les valeurs du tarif au moment de la création du séjour. Si le trésorier modifie un tarif plus tard, les factures existantes ne sont pas affectées. Ne jamais rejoindre tarif_personne pour calculer une facture — toujours utiliser les snapshots.

CREATE EXTENSION IF NOT EXISTS "pgcrypto"; -- gen_random_uuid()

-- TYPES ENUM
CREATE TYPE statut_sejour  AS ENUM ('PLANIFIE', 'EN_COURS', 'TERMINE');
CREATE TYPE mode_paiement  AS ENUM ('CHEQUE', 'VIREMENT');
CREATE TYPE statut_facture AS ENUM ('BROUILLON', 'EMISE', 'PAYEE');
CREATE TYPE statut_ligne   AS ENUM ('CONFIRME', 'A_CONFIRMER');
CREATE TYPE type_ligne     AS ENUM ('HEBERGEMENT', 'ENERGIE', 'TAXE',
                                        'SUPPLEMENT', 'COMPLEMENT_MINIMUM', 'LIBRE');
CREATE TYPE categorie_item AS ENUM ('CASSE', 'LOCATION', 'SERVICE');
CREATE TYPE unite_item     AS ENUM ('UNITE', 'SEJOUR', 'INTERVENTION');

-- ─────────────────────────────────────────────
-- CONFIGURATION SITE (clé-valeur)
-- Clés : energie_prix_nuit, energie_nb_nuits, taxe_adulte_nuit,
--        iban, delai_paiement_jours, email_resp_location
-- ─────────────────────────────────────────────
CREATE TABLE config_site (
    cle         VARCHAR(60)     PRIMARY KEY,
    valeur      TEXT            NOT NULL,
    updated_at  TIMESTAMPTZ     DEFAULT now(),
    updated_by  UUID            -- référence Cognito sub
);

-- ─────────────────────────────────────────────
-- TARIFS PAR PERSONNE (gérés par le Trésorier)
-- ─────────────────────────────────────────────
CREATE TABLE tarif_personne (
    id          UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    nom         VARCHAR(100)    NOT NULL,  -- "Membre de l'union", "Groupe de jeunes"
    prix_nuit   NUMERIC(8,2)    NOT NULL CHECK (prix_nuit >= 0),
    description TEXT,
    actif       BOOLEAN         NOT NULL DEFAULT TRUE,
    ordre       INT             NOT NULL DEFAULT 0, -- tri affichage
    created_at  TIMESTAMPTZ     DEFAULT now(),
    updated_at  TIMESTAMPTZ     DEFAULT now(),
    updated_by  UUID
);

-- ─────────────────────────────────────────────
-- ITEMS SUPPLÉMENTAIRES (gérés par le Trésorier)
-- ─────────────────────────────────────────────
CREATE TABLE config_item (
    id           UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    designation  VARCHAR(200)    NOT NULL,
    categorie    categorie_item  NOT NULL,
    prix_unitaire NUMERIC(8,2)   NOT NULL CHECK (prix_unitaire >= 0),
    unite        unite_item      NOT NULL DEFAULT 'UNITE',
    actif        BOOLEAN         NOT NULL DEFAULT TRUE,
    created_at   TIMESTAMPTZ     DEFAULT now(),
    updated_at   TIMESTAMPTZ     DEFAULT now()
);

-- ─────────────────────────────────────────────
-- LOCATAIRES
-- ─────────────────────────────────────────────
CREATE TABLE locataire (
    id          UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    nom         VARCHAR(200)    NOT NULL,
    email       VARCHAR(200)    NOT NULL,
    telephone   VARCHAR(30),
    adresse     TEXT,
    created_at  TIMESTAMPTZ     DEFAULT now()
);

-- ─────────────────────────────────────────────
-- SÉJOURS
-- ─────────────────────────────────────────────
CREATE TABLE sejour (
    id                    UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    locataire_id          UUID            NOT NULL REFERENCES locataire(id),
    statut                statut_sejour   NOT NULL DEFAULT 'PLANIFIE',
    date_arrivee          DATE            NOT NULL,
    date_depart           DATE            NOT NULL CHECK (date_depart > date_arrivee),
    nb_nuits              INT             GENERATED ALWAYS AS (date_depart - date_arrivee) STORED,
    heure_arrivee_prevue  TIME,           -- saisi par resp. location
    heure_depart_prevu    TIME,           -- saisi par resp. location
    heure_arrivee_reelle  TIME,           -- saisi par gardien
    heure_depart_reel     TIME,           -- saisi par gardien
    nb_adultes            INT,            -- pour taxe de séjour (+18 ans)
    min_personnes_total   INT             NOT NULL DEFAULT 40,
    mode_paiement         mode_paiement,
    date_limite_paiement  DATE,
    options_presaisies    TEXT,
    notes_internes        TEXT,
    created_by            UUID            NOT NULL,
    updated_by            UUID,
    created_at            TIMESTAMPTZ     DEFAULT now(),
    updated_at            TIMESTAMPTZ     DEFAULT now()
);
CREATE INDEX idx_sejour_statut_date ON sejour (statut, date_arrivee);
CREATE INDEX idx_sejour_locataire   ON sejour (locataire_id);

-- ─────────────────────────────────────────────
-- CATÉGORIES DE PERSONNES PAR SÉJOUR
-- (multi-tarifs — une ligne par tarif appliqué)
-- ─────────────────────────────────────────────
CREATE TABLE sejour_categorie (
    id               UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    sejour_id        UUID            NOT NULL REFERENCES sejour(id) ON DELETE CASCADE,
    tarif_id         UUID            REFERENCES tarif_personne(id), -- null si tarif supprimé
    nom_snapshot     VARCHAR(100)    NOT NULL,  -- COPIE — ne pas rejoindre tarif_personne
    prix_nuit_snapshot NUMERIC(8,2)  NOT NULL,  -- COPIE au moment de la création
    nb_prevues       INT,            -- saisi par resp. location
    nb_reelles       INT,            -- saisi par le gardien
    ordre            INT             NOT NULL DEFAULT 0,
    CONSTRAINT uq_sejour_tarif UNIQUE (sejour_id, tarif_id) -- 1 ligne par tarif par séjour
);
-- Note : nb_facturees N'est PAS une colonne GENERATED ici car elle dépend
-- du total de toutes les catégories (minimum global). Elle est calculée
-- en Lambda au moment de la génération de la facture.
CREATE INDEX idx_sejour_categorie_sejour ON sejour_categorie (sejour_id);

-- ─────────────────────────────────────────────
-- LIGNES DE FACTURE
-- ─────────────────────────────────────────────
CREATE TABLE ligne_sejour (
    id                  UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    sejour_id           UUID            NOT NULL REFERENCES sejour(id),
    sejour_categorie_id UUID            REFERENCES sejour_categorie(id), -- si type=HEBERGEMENT|COMPLEMENT
    config_item_id      UUID            REFERENCES config_item(id),     -- si type=SUPPLEMENT
    type_ligne          type_ligne      NOT NULL,
    designation         VARCHAR(300)    NOT NULL,  -- snapshot lisible
    quantite            NUMERIC(10,3)   NOT NULL,
    prix_unitaire       NUMERIC(8,2)    NOT NULL,
    montant             NUMERIC(10,2)   GENERATED ALWAYS AS (ROUND(quantite * prix_unitaire, 2)) STORED,
    statut              statut_ligne    NOT NULL DEFAULT 'CONFIRME',
    saisi_par           UUID,
    created_at          TIMESTAMPTZ     DEFAULT now()
);
CREATE INDEX idx_ligne_sejour         ON ligne_sejour (sejour_id);
CREATE INDEX idx_ligne_a_confirmer    ON ligne_sejour (statut) WHERE statut = 'A_CONFIRMER';

-- ─────────────────────────────────────────────
-- FACTURES
-- ─────────────────────────────────────────────
CREATE TABLE facture (
    id                    UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    sejour_id             UUID            NOT NULL UNIQUE REFERENCES sejour(id),
    numero                VARCHAR(20)     NOT NULL UNIQUE, -- FAC-2025-042
    date_generation       TIMESTAMPTZ     DEFAULT now(),
    montant_hebergement   NUMERIC(10,2),
    montant_energie       NUMERIC(10,2),
    montant_taxe          NUMERIC(10,2),
    montant_supplements   NUMERIC(10,2),
    montant_total         NUMERIC(10,2)   NOT NULL,
    statut                statut_facture  NOT NULL DEFAULT 'BROUILLON',
    pdf_s3_key            VARCHAR(500),
    email_envoye          BOOLEAN         DEFAULT FALSE,
    email_envoye_at       TIMESTAMPTZ
);

-- ─────────────────────────────────────────────
-- PAIEMENTS
-- ─────────────────────────────────────────────
CREATE TABLE paiement (
    id               UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    sejour_id        UUID            NOT NULL REFERENCES sejour(id),
    mode             mode_paiement   NOT NULL,
    montant          NUMERIC(10,2)   NOT NULL CHECK (montant > 0),
    date_encaissement DATE,
    numero_cheque    VARCHAR(50),
    banque_emettrice VARCHAR(100),
    cheque_s3_key    VARCHAR(500),
    enregistre_par   UUID            NOT NULL,
    created_at       TIMESTAMPTZ     DEFAULT now()
);
CREATE INDEX idx_paiement_sejour ON paiement (sejour_id);

-- ─────────────────────────────────────────────
-- NUMÉROTATION FACTURES (séquence par année)
-- ─────────────────────────────────────────────
CREATE TABLE facture_sequence (
    annee   INT  PRIMARY KEY,
    counter INT  NOT NULL DEFAULT 0
);

5 Calculs et vues utiles

-- VUE : Détail d'un séjour avec totaux par catégorie
CREATE VIEW v_sejour_detail AS
SELECT
    s.id             AS sejour_id,
    s.date_arrivee, s.date_depart, s.nb_nuits,
    s.min_personnes_total,
    COALESCE(SUM(sc.nb_reelles), 0) AS total_personnes_reelles,
    GREATEST(COALESCE(SUM(sc.nb_reelles), 0), s.min_personnes_total)
                     AS total_personnes_facturees,
    s.nb_adultes,
    -- Taxe de séjour
    s.nb_adultes * s.nb_nuits * (SELECT valeur::numeric FROM config_site
                                  WHERE cle = 'taxe_adulte_nuit')
                     AS montant_taxe,
    -- Forfait énergies
    LEAST(s.nb_nuits,
         (SELECT valeur::int FROM config_site WHERE cle = 'energie_nb_nuits'))
    * (SELECT valeur::numeric FROM config_site WHERE cle = 'energie_prix_nuit')
                     AS montant_energie
FROM sejour s
LEFT JOIN sejour_categorie sc ON sc.sejour_id = s.id
GROUP BY s.id;

-- VUE : Calcul hébergement par catégorie + ligne complément minimum
CREATE VIEW v_hebergement_detail AS
WITH totaux AS (
    SELECT sejour_id,
           SUM(nb_reelles)            AS total_reel,
           SUM(nb_reelles * prix_nuit_snapshot) / NULLIF(SUM(nb_reelles), 0)
                                       AS prix_moyen_pondere
    FROM sejour_categorie GROUP BY sejour_id
),
sejours AS (
    SELECT s.id, s.nb_nuits, s.min_personnes_total,
           t.total_reel, t.prix_moyen_pondere,
           GREATEST(s.min_personnes_total - t.total_reel, 0) AS ecart_minimum
    FROM sejour s JOIN totaux t ON t.sejour_id = s.id
)
-- Lignes normales
SELECT sc.sejour_id, sc.nom_snapshot, sc.prix_nuit_snapshot,
       sc.nb_reelles              AS quantite,
       sc.nb_reelles * sc.prix_nuit_snapshot * sej.nb_nuits AS montant,
       FALSE AS est_complement
FROM sejour_categorie sc
JOIN sejours sej ON sej.id = sc.sejour_id
WHERE sc.nb_reelles IS NOT NULL
UNION ALL
-- Ligne complément minimum (si nécessaire)
SELECT id AS sejour_id, 'Complément minimum contractuel',
       ROUND(prix_moyen_pondere, 2),
       ecart_minimum,
       ecart_minimum * prix_moyen_pondere * nb_nuits,
       TRUE
FROM sejours WHERE ecart_minimum > 0;

-- VUE : Éléments en attente de validation (trésorier / resp.)
CREATE VIEW v_lignes_a_confirmer AS
SELECT ls.*, s.date_arrivee, l.nom AS nom_locataire
FROM ligne_sejour ls
JOIN sejour s ON s.id = ls.sejour_id
JOIN locataire l ON l.id = s.locataire_id
WHERE ls.statut = 'A_CONFIRMER';

-- TRIGGER : maj updated_at automatique
CREATE OR REPLACE FUNCTION touch_updated_at()
RETURNS TRIGGER AS $$
BEGIN NEW.updated_at = now(); RETURN NEW; END; $$ LANGUAGE plpgsql;

CREATE TRIGGER trg_sejour_updated
    BEFORE UPDATE ON sejour
    FOR EACH ROW EXECUTE FUNCTION touch_updated_at();
-- (idem pour tarif_personne, config_item)

6 Requêtes métier principales

P1 — Séjour en cours (accueil gardien)
SELECT s.*, l.nom, l.email, l.telephone, json_agg(sc ORDER BY sc.ordre) AS categories FROM sejour s JOIN locataire l ON l.id = s.locataire_id JOIN sejour_categorie sc ON sc.sejour_id = s.id WHERE s.statut = 'EN_COURS' ORDER BY s.date_arrivee LIMIT 1;
INDEX statut, date_arrivee
P2 — Tous les tarifs actifs (création séjour)
SELECT id, nom, prix_nuit, description FROM tarif_personne WHERE actif = TRUE ORDER BY ordre, nom;
Table entière (petite)
P3 — Détail complet d'un séjour pour la facture
SELECT * FROM v_sejour_detail WHERE sejour_id = $1; SELECT * FROM v_hebergement_detail WHERE sejour_id = $1; SELECT * FROM ligne_sejour WHERE sejour_id = $1 ORDER BY type_ligne, created_at;
3 requêtes — transaction RO
P4 — Saisie gardien : maj nb_reelles par catégorie
UPDATE sejour_categorie SET nb_reelles = $2 WHERE id = $1 AND sejour_id = $3; -- sécurité UPDATE sejour SET nb_adultes = $4, updated_by = $5 WHERE id = $3;
Transaction unique
P5 — Liste séjours à confirmer (resp. / trésorier)
SELECT * FROM v_lignes_a_confirmer ORDER BY date_arrivee DESC;
INDEX PARTIEL statut = 'A_CONFIRMER'
P6 — Tableau de bord trésorier (encaissements)
SELECT EXTRACT(YEAR FROM p.date_encaissement) AS annee, p.mode, COUNT(*) AS nb, SUM(p.montant) AS total_encaisse, SUM(f.montant_total) - SUM(p.montant) AS reste_a_recouvrer FROM paiement p JOIN facture f ON f.sejour_id = p.sejour_id GROUP BY 1, 2 ORDER BY 1 DESC, 2;
Agrégat annuel
P7 — Historique locataire
SELECT s.date_arrivee, s.date_depart, s.nb_nuits, f.numero, f.montant_total, f.statut FROM sejour s LEFT JOIN facture f ON f.sejour_id = s.id WHERE s.locataire_id = $1 ORDER BY s.date_arrivee DESC;
INDEX locataire_id
P8 — Numérotation facture (transaction)
-- Dans une transaction SERIALIZABLE : INSERT INTO facture_sequence(annee, counter) VALUES($1, 1) ON CONFLICT(annee) DO UPDATE SET counter = facture_sequence.counter + 1 RETURNING 'FAC-' || $1 || '-' || LPAD(counter::text, 3, '0') AS numero;
Atomique — pas de doublon

7 Infrastructure Aurora & connexions Lambda

ComposantRecommandationJustification
Aurora PostgreSQLAurora Serverless v2 (PostgreSQL 16)Scale-to-zero la nuit / WE — usage 1×/semaine → facturation à l'usage, très économique pour ce volume
ConnexionsRDS Proxy devant AuroraLes Lambdas créent de nombreuses connexions courtes — RDS Proxy mutualise le pool et évite de saturer Aurora (max_connections)
VPCAurora + RDS Proxy en VPC privé, Lambda dans même VPCAurora n'est jamais exposé à Internet. Lambda atteint Aurora via l'endpoint du Proxy.
CredentialsAWS Secrets Manager + IAM auth RDS ProxyPas de mot de passe hardcodé. Lambda assume un rôle IAM → RDS Proxy valide via IAM → Aurora.
Driver JavaHikariCP désactivé / connexion unique par LambdaAvec RDS Proxy, pas besoin de pool côté Lambda. Ouvrir 1 connexion par invocation (ou réutiliser hors handler).
MigrationsFlyway (JAR séparé, Lambda de migration)Flyway gère le versionning du schéma. Déclenché manuellement ou via CodePipeline au déploiement.
Régioneu-west-1 (Irlande) ou eu-west-3 (Paris)Données en Europe (RGPD).
Coût estimé pour ce volume d'usage (1 séjour/semaine, ~5 utilisateurs) : Aurora Serverless v2 < 30 €/mois en production, souvent < 5 €/mois en veille. RDS Proxy : ~15 $/mois fixe. À comparer avec DynamoDB on-demand qui serait < 1 $/mois pour ce trafic — le choix PostgreSQL est guidé par la complexité métier, pas le coût.
